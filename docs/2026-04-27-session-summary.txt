PR #32  Comments hotfix — the production /home page threw a server-
        side exception (digest 596798767) immediately after #31
        because the read query used a profiles!inner nested embed
        but activity_comments.user_id FKs to auth.users, not profiles
        — PostgREST can't traverse that chain. Replaced with the
        documented two-step hydrate pattern from
        fetchLikersForActivity in lib/actions/reactions.ts (rows
        first, then a single profiles SELECT keyed by the unique
        commenter ids). Same fix applied to _addActivityComment's
        insert response.

PR #33  Comments — collapse the thread after a successful post. Adds
        an onPosted callback fired only on server-confirmed success;
        ActivityFooter wires it to setExpanded(false). Failed posts
        keep the panel open with the inline error visible.

PR #34  Comments — replaced the ✕ delete glyph on owned-comment rows
        with a small hollow uppercase "Delete" pill. The ✕ was reading
        as "close/collapse the thread" rather than "delete this
        comment."

PR #35  Comments — added a "Hide" pill to the thread panel for a more
        discoverable manual-collapse affordance. Same hollow-pill
        style as the Delete pill so the two read as a pair of utility
        pills. Wires through a new onCollapse prop.

PR #36  Comments hotfix — the Hide pill was absolutely positioned at
        the panel's top-right and overlapped the Delete pill on the
        first owned-comment row. Moved Hide into a flex header above
        the comment list so it gets its own row.

PR #37  Watch notes on the feed — when a user logs a watch with a
        note, that note now renders under the
        "<actor> watched <title>." line on /home and on profile
        pages. Migration 0132 extends activity_on_watch_insert() to
        snapshot watched.note into the activity payload at INSERT
        time. EnrichedActivity.watch_logged gains note: string |
        null. Bonus fix: profile pages were previously dropping
        watch_logged events on the floor (no case in the enrichment
        switch). Forward-only — pre-existing watch_logged rows don't
        get backfilled notes.
